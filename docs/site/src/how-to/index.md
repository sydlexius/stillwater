# How-to guides

Step-by-step procedures for the common operations. For background and *why*, see [core concepts](../core-concepts/index.md).

<div class="grid cards" markdown>

- __Run scans__

    ---

    Discover artists from disk, refresh from connected platforms, schedule recurring runs.

    [Read more](run-scans.md)

- __Edit an artist__

    ---

    Change metadata by hand, lock fields you've curated, override provider IDs.

    [Read more](edit-artist.md)

- __Filter the artists list__

    ---

    Narrow the Artists page with tri-state field, image, platform, and status filters.

    [Read more](filter-artists.md)

- __Refresh metadata__

    ---

    Pull updates from providers for a single artist or a whole library.

    [Read more](refresh-metadata.md)

- __Fetch and crop images__

    ---

    Get artwork from providers or the web, crop in-browser, save canonical filenames.

    [Read more](fetch-and-crop-images.md)

- __Fetch discography__

    ---

    Pull release groups from MusicBrainz and write them into the artist's NFO discography.

    [Read more](fetch-discography.md)

- __Configure provider priorities__

    ---

    Set which providers Stillwater asks first for each field, globally and per library.

    [Read more](configure-provider-priorities.md)

- __Enable and configure rules__

    ---

    Turn rules on and off, change automation modes, tune thresholds.

    [Read more](enable-and-configure-rules.md)

- __Export and import settings__

    ---

    Move your configuration to another instance with an encrypted bundle.

    [Read more](export-import-settings.md)

- __Update Stillwater__

    ---

    Apply a new version manually or have Stillwater check for updates.

    [Read more](self-update.md)

- __Reverse proxy__

    ---

    Front Stillwater with Caddy, Nginx, Traefik, or LSIO SWAG.

    [Read more](reverse-proxy.md)
